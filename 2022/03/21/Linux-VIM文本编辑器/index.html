Linux VIM文本编辑器

Posted on 2022-03-21 Edited on 2023-01-12 In Linux

VIM 文本编辑器使用方法。

Vim编辑器

命令 说明
vi -c n file 在第 n 行打开 file
vi -c /pattern file 在第一个出现 pattern 的地方打开 file
vi -R file / view file 以只读方式打开
vi -r 查看系统保存的文件快照
vi -r file 恢复编辑缓冲区

有时打开文件时会出现暂存盘警告，有以下几个选项：

  1. Open Read-Only 打开此文件成为只读文件，不可编辑
  2. Edit Anyway 编辑文件，但不会载入暂存文件内容
  3. Recover 加载暂存文件的内容，但在编辑保存过后还是需要手动删除
  4. Delete 删除暂存文件

一般模式

使用 vim 打开一个文件默认进入的就是一般模式。

跳转

字符跳转

命令 等价按键 功能
h ← 左
j ↓ 下（同 Enter）
k ↑ 上
l → 右（同 Space）
n + \ 光标向右移动 n 个字符
n + \ 光标向下移动 n 行
w, W (word) 光标向右移动到下一个单词词首（W 命令忽略字符及标点）
e, E 光标向右移动到下一个单词词尾（E 命令忽略字符及标点）
b，B 光标向左移动一个单词（B 命令忽略字符及标点）
(, ) 移动到当前句子开头（) 移动到结尾）
{, } 移动到当前段落开头（} 移动到结尾）
[[, ]] 移动到当前小节开头（]] 移动到结尾）

前面加数值参数可以移动多次

行跳转

命令 等价按键 功能
0 Home 光标移动到行首
$ End 光标移动到行尾
+ 到下一行行首
- 到上一行行首
^ 移动到当前行第一个字符处
n\ 移动到当前行的第 n 列

页跳转

命令 功能
H (High) 光标移动到页首
M (Mid) 光标移动到页中
L (Low) 光标移动到页尾
nH 移动到页首往下的第 n 行
nL 移动到页尾往上的第 n 行

文档跳转

命令 功能
G（Goto） 移动到文档最后一行
nG 移动到文档第n行
gg 移动到文档第一行
`` 返回到上次进行操作的地方

调整屏幕位置

命令 等价按键 功能
Ctrl + u (up) 上移半页
Ctrl + d (down) 下移半页
Ctrl + f (forward) Page Up 下移一页
Ctrl + b (back) Page Down 上移一页
z + Enter 将当前行滚动到屏幕顶端
z + . 将当前行滚动到屏幕中心
z + - 将当前行移到屏幕底部

如果前面加上数值参数，代表将指定行滚动到屏幕指定的位置。

标记

命令 功能
mx 将当前位置标记成 x
‘x 将光标移动至标记 x 所在行
`x 将光标移动至标记 x 所在字符

x 可以是任意字符

编辑模式

在一般模式下输入 i I o O a A r R 八个中的任意一个均可以进入编辑模式。

编辑模式的基本思路是：数值参数 + 操作 + 选择范围 或是 操作 + 数值参数 + 选择范围，大意是执行 n 次对选择字符的操作。

下面是插入（insert）、增加（add）、删除（delete）、修改（change）、替换（replace）、复制（yank）、粘贴（put）、连接（join）、撤销（undo）、重做（redo）。

命令 说明
i, I（insert） 从光标所在处输入（I 为从光标所在行的第一个非空格符处）
a, A（add） 从光标所在的下一个字符处输入（A 为从光标所在行的最后一个字符处）
o, O 从光标所在的下一行处输入新的一行（O 为在光标所在的上一行处输入新的一行）
d，D（delete） 删除指定范围文本（D 为删除光标至行尾的文本）
c（change） 修改，删除指定范围的文本并进入插入模式
y（yank） 复制
p，P（put） 粘贴在光标所在行之后（P 为光标所在行之前）
J（join） 将光标所在行与下一行的数据结合成同一行
u，U（undo） 撤销上一个命令（U 为恢复整行的修改操作）
[Ctrl]+r（redo） 重做上一个动作。

放置命令 p 会将缓冲区的文本放置在光标后，而执行 delete 或是 yank 操作会将内容压入默认缓冲区（共有九个位置），因此这带给我们一些全新的操作：

  • 移动：先 delete 再 put
  • 恢复：直接 put 即可恢复，通过前面添加数字编号的形式指定恢复第几个缓冲区的内容，如 3p。
  • 对调两字母：xp，先删除到缓冲区

我们也可以用 26 个专门给复制和移动文本使用的命名缓冲区（a ~ z）来进行文本的操作：

我们使用 “ 来标识缓冲区，如 "d7yy 是将 7 行文本拖至缓冲区 d。而 "dp 则是将缓冲区 d 的内容放置下来。

其他操作

命令 说明
~ 更改光标处字符大小写
. 重复上一次命令
s, S 修改光标处文本（修改整行文本）
x, X x 向后删除一个字符，X 向前删除一个字符
dd 删除光标所在行
yy 复制光标所在行
cc 修改光标所在行

常用命令

命令 说明
dd 删除光标所在行
ndd n 为数字，删除光标所在行及下方的 20 行
d1G 删除光标所在行到第一行的所有内容
dG 删除光标所在行到最后一行的所有数据
d$ 删除光标所在处到该行最后一个字符
d0 删除光标所在处到该行第一个字符
y 同理

特殊用法

对于插入模式，可以批量写入字符：通过 number(i/a/o) 命令进入插入模式，输入想要批量输入的内容，按 Esc 退出即可。

对于替换模式，可以连续替换多个字符：number(r)(character) 即可将多个连续字符替换为期望字符。

替换模式

通过键入 r 可取代光标所在的字符一次，而通过 R 会直接进入替换模式。

块选择模式

按键 意义
v（小写） 字符选择，选择光标经过的地方
V（大写） 行选择，选择光标经过的行
[Ctrl]+v 块选择，可以用长方形的方式选择数据
y 将选择的内容复制
d 将选择的内容删除

命令行模式

在一般模式下输入 : 、/、? 三个中的任意一个均可以进入命令行模式。

操作

命令行模式的基本思路是：行号/行号范围（默认为当前行） + 操作。

行寻址符号：. 表示当前行，0 表示文件第一行，$ 表示文件最后一行，% 表示文件中的每一行（等于 1,$）。

操作：m 移动，d 删除，co/t 复制

命令 功能
:m 跳转到第 m 行
:m,n 跳转到第 m,n 行
:2s/apple/banana 将第 2 行的 apple 字符串替换成 banana 字符串
:1,$d 删除 1 到 最后一行
:.,4m15 将当前行到第 4 行移动到第 15 行之后
:1,4co15 (co = t) 将 1 到 4 行复制到第 15 行之后
:%t$ 将所有行复制到文件结尾

我们还可以对行号进行计算：如：

.,.+20m.-2 将当前行到往下 20 行的内容向上移动两行

但也可以忽略 .，因为 + 和 - 是默认当前行的。于是上面的命令可以简化为 ：

.,+20m-2

参数定义

命令 功能
:set number / :set nu (:set nonumber / :set nonu) 显示行号（取消显示行号）
:set wm（wrap margin） 设置一行包含的字符数
:set hlsearch（:set nohlsearch） 高亮搜索关键词（取消高亮关键词）
:set auotoindent（:set noauotoindent） 设置自动缩排（取消自动缩排）
:syntax on（:syntax off） 语法高亮
:set bg=dark（:set bg=light） 背景色设置
:set all 显示所有环境参数设置
:set 显示与系统默认值不同的环境参数设置

通过在 ~/.vimrc 文件设置 vim 的环境变量来使配置永久生效

搜索

全局搜索

命令 功能
/pattern 从光标位置向下搜索
?pattern 从光标位置向上搜索
n 往同一个方向重复搜索
N 往相反方向重复搜索
/ + Enter 向前重复搜索
? + Enter 向后重复搜索

由于搜索会将光标移动至匹配单词的第一个字符，因此可配合操作实现增删改等功能。

行搜索

命令 功能
fx 移动到本行下一个出现 x 的地方
Fx 移动到本行上一个出现 x 的地方
tx 移动到本行下一个出现 x 的地方的前一个字符
Tx 移动到本行上一个出现 x 的地方的前一个字符
; 重复上一个搜索命令，方向相同
, 重复上一个搜索命令，方向相反

x 代表任何字符

查找

命令 功能
:g/pattern 显示全局包含 pattern 的文本
:g!/pattern 显示全局不包含 pattern 的文本
:g/pattern/nu 显示全局包含 pattern 的文本并显示行号
:60,100g/pattern 显示 60 到 100 行包含 pattern 的文本

替换

行替换（rows substitute）

替换有三个关键词 s substitute（替换）、g global（替换整行）、c confirm（对替换的内容进行确认）。

命令 功能
:s/pattern/content/ 将当前行的匹配到的第一个 pattern 换成 content
:s/pattern/content/g 将当前行的匹配到的全部 pattern 换成 content
:50,100s/pattern/content/g 将当前 50,100 行的全部 pattern 换成 content
:1,$s/pattern/content/g 将全文的全部 pattern 换成 content
:%s/pattern/content/g 同上，% 与 1,$ 等价
:%s/pattern/content/gc 对替换的内容进行确认

全局替换（global substitute）

命令 功能
:g/pattern/s/old/new/g 全局匹配包含 pattern 的行，并将 old 替换为 new
:g/pattern/s//new/g 如 pattern 和 old 相同可省略

保存和离开文件

命令 功能
ZZ 若文件未改动则不保存离开；若文件已经被改动，则保存后离开
:e 放弃所有修改，回到原来文件
:q 离开编辑器
:q! 强制离开编辑器，并放弃所有修改
:w 将缓冲区内容写入文件
:w! 若文件属性为只读时，强制写入该档案。能否正确写入与权限有关
:w [filename] 将编辑的数据储存成另一个文件
:w %.new % 表示当前文件名，可以另存为一个新文件
:n1,n2w [filename] 将 n1 到 n2 的内容储存成 filename 这个档案
:n1,n2w >> [filename] 将 n1 到 n2 的内容附加到现有的 filename 文件结尾
:x 将缓冲区内容写入文件，并离开编辑器
:read [filename] / :r [filename] 在编辑的数据中，读入另一个档案的数据
:17r [filename] 在第 17 行后读入另一个文档的数据
:/pattern/r [filename] 将读入的文件放在第一个出现在 pattern 的行之后

常用命令

命令 功能
:= 列出文件总行数
:.= 列出当前所在行行号
:/pattern/= 列出 pattern 第一次出现的行号
:! command 暂时离开 vi 到指令行模式下执行 command 的显示结果

多文件编辑

vi file1 file2 同时打开多个文件并使用多文件编辑功能。通过正常模式下用到的缓冲区来传输内容。

命令 说明
Ctrl + ^ 切换文件
:n 编辑下一个文件
:N 编辑上一个文件
:files 列出 vim 打开的所有文件

多窗口功能

命令 说明
:sp [filename] 打开一个新窗口
[Ctrl] + w 切换窗口
[Ctrl] + w + j / [Ctrl] + w + ↓ 光标跳向下一个窗口
[Ctrl] + w + k / [Ctrl] + w + ↑ 光标跳向下一个窗口
[Ctrl] + w + q / :q 关闭此窗口

[Ctrl] + w + 的按法，先按 [Ctrl] + w，松开后按 的键即可。

注意事项

中文编码的问题

如果使用 VIM 乱码可能需要考虑：

  1. Linux 系统支持的语系数据：/etc/sysconfig/i18n
  2. 终端接口 bash 语系，这与 LANG 变量有关
  3. 文件原本的编码
  4. 打开文件的软件

语系编码的转换可以使用 iconv 命令。
