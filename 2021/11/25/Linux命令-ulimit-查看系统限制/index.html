Linux命令 ulimit 查看系统限制

Posted on 2021-11-25 Edited on 2022-09-08 In Linux

ulimit 为 Shell 内建指令，可用来控制 Shell 执行程序的资源。

语法

ulimit [-aHS][-c <core文件上限>][-d <数据节区大小>][-f <文件大小>][-m <内存大小>][-n <文件数目>][-p <缓冲区大小>][-s <堆叠大小>][-t <CPU时间>][-u <程序数目>][-v <虚拟内存大小>]

参数

  • -a 　显示目前资源限制的设定。
  • -H 　设定资源的硬性限制，也就是管理员所设下的限制。
  • -c <core文件上限> 　设定core文件的最大值，单位为区块。
  • -d <数据节区大小> 　程序数据节区的最大值，单位为KB。
  • -f <文件大小> 　shell所能建立的最大文件，单位为区块。
  • -m <内存大小> 　指定可使用内存的上限，单位为KB。
  • -n <文件数目> 　指定同一时间最多可开启的文件数。
  • -p <缓冲区大小> 　指定管道缓冲区的大小，单位512字节。
  • -s <堆叠大小> 　指定堆叠的上限，单位为KB。
  • -S 　设定资源的弹性限制。
  • -t <CPU时间> 　指定CPU使用时间的上限，单位为秒。
  • -u <进程数目> 　用户最多可开启的进程数目。
  • -v <虚拟内存大小> 　指定可使用的虚拟内存上限，单位为KB。
  • -i <信号挂起数>　指定信号可以被挂起的最大数。

实例

显示系统资源的设置

[root@elijah ~]# ulimit -a
core file size     (blocks, -c) 0
data seg size      (kbytes, -d) unlimited
file size        (blocks, -f) unlimited
pending signals         (-i) 1024
max locked memory    (kbytes, -l) 32
max memory size     (kbytes, -m) unlimited
open files           (-n) 1024
pipe size      (512 bytes, -p) 8
POSIX message queues   (bytes, -q) 819200
stack size       (kbytes, -s) 10240
cpu time        (seconds, -t) unlimited
max user processes       (-u) 4096
virtual memory     (kbytes, -v) unlimited
file locks           (-x) unlimited

提高文件句柄限制

Linux 系统限制了文件打开数目，例如分表太多可能会导致超出 Linux 默认限制。

通过下述命令可以查看当前进程可打开的最大文件数。

ulimit -a | grep "open files"
ulimit -n

相对于当前线程，系统也有对应的可打开的最大文件数，这个值代表所有进程的总和上限。

less /proc/sys/fs/file-nr

打开文件会看到三个值，分别是：系统已分配文件句柄的数目、系统已使用文件句柄的数目、系统可打开文件句柄的最大数目。该文件是只读文件，只用来展示信息。也可以通过查看 /proc/sys/fs/file-max 文件来获取系统可打开文件句柄的最大数目的属性信息。

通过命令 ulimit -n 65535 可以只修改当前进程的可打开最大文件数，但重启会失效。通过向 /etc/security/limits.conf 添加以下内容，可以永久生效。

* soft    nofile    65535
* hard    nofile    65535

而通过修改 /proc/sys/fs/file-max 可以更改整个系统的最大的文件数，其内容的修改仍然只是临时生效。通过向文件 /etcsysctl.conf 添加 fs.file-max=655360 就可永久生效。

这里的 limit 分为 soft limit 和 hard limit， soft limit 不能超过 hard limit，而普通用户只能修改 soft limit，或者降低 hard limit。

需要注意的是：

  1. 所有进程打开的文件描述符数不能超过 /proc/sys/fs/file-max
  2. 单个进程打开的文件描述符数不能超过 nofile 的 soft limit
  3. nofile 的 soft limit 不能超过 hard limit
  4. nofile 的 hard limit 不能超过 /proc/sys/fs/nr_open
